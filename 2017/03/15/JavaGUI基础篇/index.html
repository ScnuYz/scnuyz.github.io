Java GUI

Java GUI 包括哪两块技术体系？

AWT

Swing

AWT

在JDK1.0的时候，Java就提供了这一套基本的GUI类库给开发者

当使用AWT编写图形界面的时候，仅指定了界面组件的位置和行为，未提供真正的实现。

JVM只是在运行的时候调用了操作系统本地的图形界面来创建和平台一致的对等体

从而导致了一些问题：

  1. 依赖本地平台，而且是重量级组件，消耗大
  2. 为了实现可移植性，只能构造多个操作系统共有的界面模块，导致提供的功能有限，不能使用复杂的界面组件，而且不同系统有不同表现，例如同一个程序在windows中运行的界面和在unix中运行的界面不同

Swing

Swing完全使用Java实现界面组件，所以有如下特点

  1. 速度比AWT慢，但是对于现在的硬件设备来说，速度的区别不明显
  2. 轻量级组件消耗小
  3. 不依赖平台，而且提供了大量的图形界面组件，功能丰富，在不同的平台有相同的表现

主要需要掌握的？

布局、组件、事件

Java GUI 的布局

Java提供了哪些常用布局管理器和特殊的布局容器？

常用布局管理器：

  1. FlowLayout
  2. BorderLayout
  3. GridLayout
  4. CardLayout
  5. 绝对定位

特殊布局容器：

  1. Swing特殊容器：Box
  2. Swing特殊容器：JLayeredPane

BorderLayout

JFrame默认的布局管理器就是BorderLayout

这个布局管理器吧容器分为了东西南北中

也就是EAST SOUTH WEST NORTH CENTER

然后其他的组件就可以放在这被放置在容器的指定区域，但是默认是center

同一个区域只能放一个组件，后者会把前者覆盖，就像上面第一个例子那样，second button覆盖了first button

所以一个BorderLayout容器最多只能放5个组件

如果某个区域没有放置组件的话，那么旁边的组件就会自动占据这片区域，不出现空白，就像上面第一个例子那样，没有留空白

GridLayout

GridLayout将容器分割成从横交错的网格

每个网格所占的区域大小相同，往容器中添加组件的时候，默认从左向右，从上向下，依次添加到网格中

组件的大小由网格的大小决定，即是会占满整个网格

这里要注意一下：

如果向容器里面添加的组件数目少于或多于网格数的时候，容器会按所设置的行数固定行数，然后自动调节列数

假如，设置4行2列，添加3个组件

同样的，他也可以设置间距，如果不了解可以直接查询API文档

CardLayout

设置CardLayout的容器，里面的组件就像卡片一样全部叠在了一起

然而我们每次只能看到最顶层的卡片

通过CardLayout提供的方法，可以吧其他的卡片叠到顶层并显示出来

绝对定位

把一个容器设成了绝对定位的布局，里面的组件必须定好位置与大小

特殊布局容器：Box

Box容器可以使得组件在水平或垂直方向上一一排列

水平Box类似于FlowLayout容器，但不会折回下一行

特殊的布局容器：JLayeredPane

他是一个层叠容器，允许容器中的组件层叠起来，但是效果与CardLayout容器不同

注意：

  1. JLayeredPane采用的也是绝对定位的布局，里面的组件同样需要定义大小与位置
  2. 往里面添加组件的时候需要指定一个层次关系的参数，通常使用Integer对象（并不是int），数值越大越置顶

事件

什么是事件？

也就是用户操作，当点击某个按钮，文本输入等能触发事件。

Java GUI提供了多个事件类来表示各种事件

例如

  1. WindowEvent表示窗口事件
  2. KeyEvent表示键盘事件
  3. MouseEvent表示鼠标事件
  4. ActionEvent动作事件（比较常用，例如按钮被点击，输入框按Enter）

什么是事件绑定？

当用户进行操作的时候，系统需要作出什么响应，就要绑定到对应组件的事件上。

绑定（或称为监听）组件事件，就需要用到事件监听器

例如

  1. WindowListener
  2. KeyListener
  3. MouseListener
  4. ActionListener

那么问题来了，怎么使用事件监听器进行事件的绑定呢?

如何使用事件适配器进行事件绑定？

Java为每个定义多方法的监听器接口都提供了一个实现类，称为事件适配器，他把对应的监听器接口都提供了空的实现，我们使用适配器进行事件绑定时，只要用匿名内部类的方式创建适配器子类并且重写我们需要关注的方法即可

点击关闭后console显示：关闭程序…

为什么事件绑定通常使用匿名内部类？

因为每个组件对应事件的实现都是不同的！！
