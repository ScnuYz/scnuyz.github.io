XML基础

XML是什么？有什么特点？

XML，可以理解成是一种格式规范，让数据结构更清晰！

XML（Extensible MarkupLanguage），即可扩展标记语言，它与HTML一样，都是SGML（StandardGeneralized Markup Language，标准通用标记语言）。P.S：HTML历史更早

XML最主要的作用是存储数据，最主要的特点是可读性高。

  • 与数据库的不同

XML，仅仅是一种标记语言，更简单，只用于存储数据；
数据库，是一个软件，更强大，除了存储数据，还有更多的功能。

  • 与HTML的不同

XML，仅仅是一种标准的数据格式，用于存储数据；
HTML，数据格式标准+Web浏览器标准，结合浏览器，用于数据的显示。

XML，由于它的简单易用、可读性高，很快成为一个通用的标准，独立于各种语言技术体系和平台。

以文本文件的方式存储数据

数据交换（如系统集成、数据接口、AJAX等）

通过XML，可以在不兼容的系统平台、不同的程序之间交换数据。譬如：

A医院财务处使用用友财务系统（U8），而门诊挂号、计价收费等使用医院信息管理系统（HIS），每个月，财务处需要将HIS系统中的收费信息整理统计，录入到U8系统中，手工操作导致很大的工作量。因此，希望能够通过软件自动/一键导入的方式，从HIS中把数据导入到 U8中。

首先，U8的数据库不会让其它系统操作，HIS也是一样。

双方讨论的解决方案是：HIS开发一个数据导出模块，将数据导出到一个文本文件中；U8开发一个数据导入模块，将HIS导出的文本文件解析导入到自己的数据库中。

这两个模块之间，如何进行数据交换呢？使用“, , ;”那样的标记？

通用的XML会是最好的选择，双方只需要定义好XML的结构标记（“协议”）即可。

如何定义一个XML文档

注意以下规范：

  1. 第一行是XML序言

version，表示规范版本，一般为1.0；
encoding，表示解析器解析时使用的编码（应与文件编码一致）；

  1. 最外层必须有一个根节点
  2. 任何的起始标签都必须有一个结束标签，“/”**不能漏**
  3. 可以采用简化语法，解析器解析时会将其翻译成<tag></tag>

因此，表示书的价格可有两种方式：<price value="56.0" />和<price>56.0<price/>，但是解析的方式不同。
建议：元数据（有关数据的数据，如id）应当存储为属性，而数据本身应当存储为元素。

  1. 标签必须按对应顺序嵌套
  2. 标签大小写敏感
  3. 所有的属性都必须有值，即不能忽略“=”及后面的值；值的两边必须加上引号，单引号与双引号都可以，但一般建议使用双引号（比HTML严格）
  4. 注释同HTML
  5. 特殊字符问题（实体引用）
  6. 命名规则：名称可以包含字母数字以及其他字符，名称不能以数字或者标点符号开始，名称不能以xml开始，名称不能包含空格，它没有保留字词

解析XML的基本方式有哪些？在不同语言中有何区别？

解析XML的基本方式有两种：

一种叫DOM，是基于XML文档树结构的解析；

另一种叫SAX，是基于事件的解析。

这两种方式是两种解析思路，在不同的语言中都是一样的，只是实现的语法不同而已。

DOM和SAX的特点与应用？

  • DOM

解析器读取整个XML文档，构建一个驻留内存的树结构，然后就可以使用DOM接口来操作这个树结构。
优点：整个文档树在内存中，便于操作，支持随意读取、删除、修改、排列等多种功能；
缺点：读取整个文档到内存中（包括不需使用的节点），浪费时间和空间；
使用场合：文档解析后需多次访问/操作其中的数据；硬件资源充足（内存、CPU）；
应用：见案例

输出：

  • SAX

为解决DOM的问题， SAX出现了。SAX，基于事件解析，当解析器发现元素开始、元素内容、元素结束等时候，触发事件，程序员编写响应代码，保存/处理数据。
优点：不用事先读取整个文档，也可“提前”结束读取，占用资源少，速度快；
缺点：无状态性，事件过后，如果没有保存数据，那么数据就丢失；从事件中只能得到文本，但不知该文本属于哪个元素；
使用场合：只需读取XML文档中的部分内容，很少反复访问与操作；硬件资源不足；
应用：见案例（现在一般使用SAX2，第二版标准）

输出：

Java官方对DOM和SAX两种方式提供了很好的支持；

另外，还提供了基于XML流的解析方式StAX（类似于SAX，可理解成是SAX模式的扩展）。
DOM、SAX、StAX组成了JAXP（Java API for XML Processing）标准的3大解析方式。

JDOM和DOM4J是什么？

DOM和SAX是两种标准的解析方式，API提供的都是一些比较底层的方法，因此使用的时候难免比较复杂，代码量较大。

于是，Java这边就有人专门开发Java专用的XML工具API，以便减少编码量，更好、更方便地解析、操作XML，其中，最常用的两大组件就是JDOM和DOM4J。

JDOM和DOM4J，不仅仅是单纯的DOM组件，可以理解为，是基于JAXP标准与更多XML相关的应用支持，进行不同程度的封装、优化、扩展而成的两个组件。之所以以“DOM”为名，是因为它们解析XML都生成对应的树，但是，它们都采用SAX解析器，从而可以同时进行SAX事件控制。
其中，DOM4J的性能更加优秀，目前很多的Java框架都应用它进行XML的解析/操作，因此，更加推荐使用。

DOM、SAX、JDOM和DOM4J的对比？

  • DOM

基本解析方式，不限于Java，读取的时候同时在内存中生成树，时间主要损耗在树的生成，内存占用大。

  • SAX

基本解析方式，不限于Java，基于事件解析，速度快，时间主要损耗在事件监听上，内存占用情况自控。

  • JDOM

Java组件，生成树的同时，采用SAX解析器，可进行SAX事件监听；易用，且由于结合两者，功能更加强大；但是一般来说，解析速度会比原生的DOM和SAX慢（如案例中的方式读取大文件时会非常慢），内存占用情况同DOM。

  • DOM4J

同JDOM，但性能更优秀，且在大文件读取内存占用上提供对应方案（detach
node），一般情况推荐使用。

解析速度比较：SAX> DOM > DOM4J > JDOM

XML与JAVA对象的转换？

  • JAXB

Java Architecture for XML Binding，Java官方提供的一个标准，功能强大。

  • XStream

第三方工具组件，应用起来相对更加简单。

JS一般使用DOM方式解析XML。

JAVASCRIPT对XML的解析？

JS一般使用DOM方式解析XML。

为了兼容性，最好使用JQuery进行XML相关操作：
JQuery Ajax，加载XML文档文件；
JQuery.parseXML()方法，把XML字符串转换成XML对象；
JQuery节点操作。
